# Networking

This discovery domain explains how the Azure Local environment is physically and logically connected.

## Scope

It should capture:

- virtual switches and SET configuration
- host vNICs and IP configuration
- management, storage, and compute networking
- RDMA and DCB-related posture
- Network ATC intents and status
- SDN components where deployed
- logical network constructs tied to the deployment
- DNS, proxy, and firewall context

## Why It Matters

Networking is one of the main ways an inherited Azure Local environment becomes hard to understand. Ranger should make that design legible.
