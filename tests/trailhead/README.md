# TRAILHEAD

This directory contains the AzureLocalRanger live field-validation solution.

## Contents

- `field-testing.md` — the Operation TRAILHEAD live-validation methodology and phase model
- `scripts/` — helpers for starting runs and creating GitHub phase cycles
- `logs/` — committed run logs from live validation sessions
- `docs/` — detailed TRAILHEAD documentation

## Scope

Everything under `tests/trailhead/` is part of live validation against a real environment.

Offline synthetic testing, fixture-backed Pester coverage, and cached-manifest render validation live under `tests/maproom/`.

`repo-management/` remains reserved for repository governance, planning, and audit material.
