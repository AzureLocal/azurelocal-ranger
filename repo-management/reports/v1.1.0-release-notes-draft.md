# Azure Local Ranger v1.1.0 Release Notes Draft

Status: validated and ready for milestone closeout

## Summary

This milestone focuses on post-0.5.0 operational hardening and documentation depth for Azure Local Ranger. The core outcome is that Ranger can now run more safely in scheduled workflows, compare a current discovery package against a prior baseline, and surface materially better storage and Arc guest insights in its reports.

## Highlights

- Scheduled and unattended execution via `-Unattended`, including `run-status.json` for job monitoring.
- Baseline manifest comparison via `-BaselineManifestPath`, including `manifest/drift-report.json` and drift reporting in technical output.
- Deep Storage Spaces Direct capacity analysis with reserve modeling, safe allocatable headroom, and thin-provisioning findings.
- Arc guest-IP fallback for VMs where Hyper-V data exchange did not provide guest addresses.
- Arc ESU eligibility and enrollment assessment for supported Windows Server guests.
- Contributor-facing technical runtime flow diagram and update-mode design notes.
- Completed TUI alternatives survey with `PwshSpectreConsole` selected as the preferred rich-terminal direction.

## Operator Impact

- Operators can now run Ranger non-interactively in Task Scheduler or CI with clearer failure signaling.
- Repeated runs can be compared against an approved baseline manifest without external tooling.
- Technical and management reports carry stronger storage-capacity guidance, including reserve targets and safe growth headroom.
- Arc-connected guest reporting is more accurate when Hyper-V guest services do not return IP data.
